blur/focusout切換畫面不斷被觸發的解決方法

發表於 2017-04-29 |

情況是這樣的，我們有一個輸入框，並綁定了 focusout 事件，當這個事件發生的時候，會做些處理然後跳alert說明處理結果。

OK，這一切都很順利很美好。

直到我們測試出了當使用者點了輸入框，然後切換視窗之後，那就會不斷地跳出alert!! (這時候就是比手速了)

直接說結論，在alert 之前 focus 其他元件。

我利用 jsFiddle 重現了這個現象與旁門左道式的解法。 (如果想比手速，可以把$("#outFocusInput").focus()註解掉 )。

觀察與解決歷程

新手如我，下了關鍵字查詢(blur + chrome + trigger + times + ...)，換了blur，加了 return false; 或者是event.preventDefault()，我的瀏覽器依然不斷耍任性。

後來發現，當我點掉提示訊息的時候，輸入框裡面還是有提示字元，代表說我點完了彈跳訊息之後，我又回到了那個輸入框，可是又不斷的 focusout / blur。

所以，我就把在 alert 之前，就先 focus 其他元件就好。 如果很不幸的，這個頁面除了這個輸入框以外沒有其他朋友可以讓我們focus的，就創一個並移出畫面之外。
