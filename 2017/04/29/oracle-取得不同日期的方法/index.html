oracle-取得不同日期的方法

發表於 2017-04-29 |

簡單介紹一下 Oracle 日期取得方式，這篇的內容包含了:

  1. 取得今天的日期
  2. 取得這周的禮拜日與禮拜六
  3. 取得這個月的第一天與最後一天
  4. 列出這個月每一天 (利用connect by)
  5. 綜合練習: 列出當月完整的日期(這個月第一天)

這篇主要會用到的函數包含了:

  • sysdate: 回傳今天日期。
  • trunc(date, [fmt]): 回傳一個日期，為傳入參數的 Date ，截斷 fmt 單位以下的日期部分。
    • 如果 fmt 沒有指定 ，預設為 'DAY'
    • 如果 fmt = 'WW'，以結果來說，可以取得這星期的第一天。
    • 如果 fmt = 'Y'，
  • last_day(date): 取得該月最後一天。
  • add_months(date, months): 取得 months 個月後的日期。
  • connect by level <= n : - 迴圈查詢的方法，這邊的用法，可以把level 當作我們寫程式用到 for 迴圈的用法，就當作 for(int level = 1; level <= n; level ++){}

取得今天的日期: (sysdate)

  • 需要注意的是，oracle 裡面的 sysdate 回傳的雖然是日期型態(Date)，但是其實有包含時分秒。
  • 換句話說，如果要拿來跟其他時間比較時，可以使用 trunc() 去操作，如果沒有預設值的話，會捨去 hour以下的數值，包含hour。

取得這周的禮拜日與禮拜六

  • 日期直接 + 1 就是隔天了。換句話說， - 1 就是昨天。
  • 所以邏輯是，取得了這禮拜的第一天(禮拜日)，加上指定日數，就可以取得禮拜幾了。

取得這個月的第一天與最後一天

  • 取得最後一天的第一種作法，取得下個月的第一天再減一。
    • 這主要來自 sql server 比較舊版本(2008 R2 以下)的邏輯，新版本有EOMONTH()

列出這個月每一天

  • 如上面提到的，可以使用 connect by level <= n 去跑迴圈。
  • 先計算好這個月的天數，然後丟給level。注意要 + 1，不然結果會少一天。
  • 再依據這個月第一天去加上 level 的數字。記得這邊要 - 1，不然會位移一天。
